<h3>Terry-Mao/gopush-cluster Web http协议文档</h3>
Web内部接口文档，用于推送消息、节点管理等

<h3>接口汇总</h3>
(head). | 接口名 | URL | 访问方式 |
| "推送私有消息":AdminPush | /admin/push     | POST |
| "添加节点":AdminNodeAdd | /admin/node/add | POST |
| "删除节点":AdminNodeDel | /admin/node/del | POST |
| "清理消息":AdminMsgClean | /admin/msg/clean | POST |

<h3>公共返回码</h3>

(head). | 错误码 | 描述 |
| 0 | 成功 |
| 65534 | 参数错误 |
| 65535 | 内部错误 |

<h3>基本返回结构</h3>
<pre>
{
    "ret": 0,  //错误码
    "data": {….}//如果有数据返回，则有data，无数据返回就只有前面的ret和msg
}
</pre>

<h3>推送私有消息</h3>
 * 请求参数

(head). | 参数 | 类型 | 描述 |
| key    | string | 订阅key |
| gid    | string | Group ID (写死0即可)|
| expire | int64  | 消息过期时间，单位：秒(s)|
注: 消息体存放到body中，以上参数为URL参数

 * 返回码

(head). | 错误码 | 描述 |
| 1001 | 没有找到key对应的节点 |
<pre>
{
    "ret": 0
}
</pre>

<h3>添加节点</h3>
 * 请求参数

(head). | 参数 | 类型 | 描述 |
| node | string | 节点名称 |

 * 返回码

<pre>
{
    "ret": 0
}
</pre>

<h3>删除节点</h3>
注：删除节点不会主动断开该节点与客户端的订阅长链接
 * 请求参数

(head). | 参数 | 类型 | 描述 |
| node | string | 节点名称 |

 * 返回码

<pre>
{
    "ret": 0
}
</pre>

<h3>清理消息</h3>
注：清理单个订阅(key)下的所有消息，并从Comet模块中清理掉Key对应的Channel
 * 请求参数

(head). | 参数 | 类型 | 描述 |
| key | string | 客户端订阅时的key |

 * 返回码

<pre>
{
    "ret": 0
}
</pre>
